El Vibe Check

La velocidad es real. El riesgo también.

  • Prototipos y features salen en horas, no semanas.
  • Sin barandales, la deuda y el drift arquitectónico se disparan.
  • La confianza del equipo baja cuando "funciona" pero nadie entiende el porqué.

Los 3 pecados más comunes

  • Pecado #1: mandar a producción sin leer ni entender el código.
  • Pecado #2: fallas silenciosas por falta de observabilidad.
  • Pecado #3: ausencia de specs; el código termina siendo la única “documentación”.

Barandales prácticos

  • Regla dorada: si no puedes explicar el código, no lo comitees.
  • Flujo recomendado: Spec → Prompt → Generate → Test → Review → Ship.
  • Automatiza calidad: lint, type-check y security scan en cada guardado/CI.

Nuevo rol del desarrollador

  • Menos escritura de sintaxis, más diseño de sistemas.
  • Menos velocidad de tipeo, más claridad de intención.
  • El valor cambia de “líneas entregadas” a “problemas resueltos”.

Qué hacer este lunes

  • Audita código generado por IA para detectar intención perdida.
  • Formaliza la regla: “si no lo puedes explicar, no lo subes”.
  • Agrega al menos un guardrail automático a tu pipeline hoy.
  • Identifica una tarea commodity para delegar a agentes IA.
