1) Reality check: dónde sí brilla y dónde se rompe

Vibe coding: construir software conversando con IA (intención) en lugar de escribir cada línea manualmente (sintaxis).

Lo que sí funciona

  • Velocidad brutal para prototipos y primeras versiones.
  • Un individuo puede ejecutar como un equipo pequeño.
  • De semanas a fines de semana para lanzar demos funcionales.

Lo que explota después

  • Regresiones en cascada: arreglas 1 cosa y rompes 10.
  • Errores de producción: 429, 504, bucles infinitos, OOM.
  • Rollback fallido por falta de estado/versionado claro.

3) Los 3 pecados de equipos con IA

  • Pecado #1 — Enviar sin entender: Probabilístico ≠ determinístico. La IA puede alucinar APIs y lógica.
  • Pecado #2 — Fallo silencioso: sin observabilidad te enteras por el cliente, no por el sistema.
  • Pecado #3 — Intento perdido: sin specs, solo queda código opaco; imposible refactor seguro y onboarding lento.

Ejemplo clásico: funciones "mágicas" sin contexto de negocio ni justificación técnica.

4) Guardrails para entregar rápido sin suicidar producción

"Si no puedes explicar el código, no lo comitees." — Simon Willison

Workflow que pierde

  • Prompt → Generate → Ship → Pray

Workflow que escala

  • Spec → Prompt → Generate → Test → Review → Ship
  • Heurística: si puedes escribir unit test, puedes "vibear"; si no, primero spec.
  • Shift-left: lint + type-check + security scan antes de revisión humana.
  • Costo: corregir en loop (~$0.10) vs corregir en prod (~$10,000).

5) Nuevo leverage del developer (2026)

  • Ya no te pagan por teclear; te pagan por pensar y diseñar sistemas.
  • Bottleneck viejo: velocidad de escritura / memoria de APIs.
  • Bottleneck nuevo: claridad de intención, arquitectura y criterio.
  • Métrica vieja: líneas enviadas. Métrica nueva: problemas resueltos.

De bricklayer a foreman. El rol sube de ejecución mecánica a dirección técnica.

6) El reset SaaS: vender outcomes, no solo herramientas

  • Modelo viejo: "renta una app" (valor = tiempo en producto).
  • Modelo nuevo: "contrata un agente" (valor = trabajo completado).
  • Caso ideal: prompt a producción en 45 min con 0 líneas manuales y 0 bugs en prod.

7) Lunes en la mañana: plan accionable

  • Audita código reciente de IA para detectar "intent loss".
  • Impón la regla dorada en PRs.
  • Agrega 1 guardrail automático a CI esta semana.
  • Elige 1 habilidad commodity para delegar a IA (boilerplate, glue code, etc.).
